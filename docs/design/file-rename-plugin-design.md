# File-Rename 插件设计文档

## 1. 功能概述

**File-Rename** 是一个批量重命名文件的内置插件，支持多种重命名规则，以及拖拽排序和手动编辑。所有操作均在用户确认后才真正写入磁盘，保证安全性。

| 项目 | 值 |
|---|---|
| 插件 ID | `file-rename` |
| 分类 | `file` |
| 图标 | `✏️` |
| 插件路径 | `plugins/builtin/file-rename/` |

---

## 2. 功能规格

### 2.1 文件输入

支持两种方式，可混合使用：

- **拖拽**：将文件或文件夹拖入预览区的拖放区域
- **按钮选择**：点击「选择文件」打开多选文件对话框；点击「选择文件夹」打开文件夹对话框

> 若输入的是文件夹，只列出其**第一层**文件，不递归进入子目录；子目录本身不列入列表。

文件加入列表后，默认按**文件名字母顺序**排序。重复添加同路径文件时，自动去重忽略。

---

### 2.2 重命名规则

每次只能选择一种规则（单规则模式）。规则作用于预览列表中所有文件，生成「新文件名」列的预览值。

#### 规则一：查找替换

| 参数 | 说明 |
|---|---|
| 查找 | 输入要匹配的子字符串（大小写敏感，普通字符串匹配，不支持正则） |
| 替换为 | 输入替换后的字符串（可为空，表示删除匹配内容） |

- 若文件名中存在多处匹配，全部替换
- 若文件名中无匹配，该文件新名 = 原名（无变化，预览中标灰提示）

#### 规则二：前后缀

| 参数 | 说明 |
|---|---|
| 前缀 | 添加在文件名（不含扩展名）最前面的字符串，可为空 |
| 后缀 | 添加在文件名（不含扩展名）最后面、扩展名之前的字符串，可为空 |

示例：原名 `photo.jpg`，前缀 `vacation_`，后缀 `_2026` → `vacation_photo_2026.jpg`

#### 规则三：序号模式

将**整个文件名（不含扩展名）**替换为纯序号，保留原扩展名。

| 参数 | 说明 |
|---|---|
| 起始数字 | 序号从此值开始，默认 `1` |
| 是否补零 | 开关，关闭则不补零；开启则自动计算位数（按列表总数）补零 |

示例（10 个文件，起始 1，补零开启）：`001.jpg`, `002.jpg`, ..., `010.jpg`  
示例（3 个文件，起始 5，补零关闭）：`5.jpg`, `6.jpg`, `7.jpg`

> 序号顺序与预览列表中的排列顺序一致，拖拽排序直接影响序号分配。

#### 规则四：手动编辑

在预览列表中，直接点击任意一行「新文件名」单元格进行行内编辑。

- 手动编辑可与其他规则**叠加**：先选择某规则生成全部新名，再对个别文件手动微调
- 切换规则时，已手动修改的行会被新规则覆盖（切换前弹出提示确认）

---

### 2.3 预览列表

| 列 | 说明 |
|---|---|
| 拖拽手柄 | 左侧图标，用于 SortableJS 拖拽排序 |
| 原文件名 | 不可编辑，灰色 |
| 新文件名 | 由当前规则计算得出；手动编辑模式下可点击行内编辑 |
| 状态标记 | 无变化（灰色「-」）/ 有变化（蓝色）/ 命名冲突（红色警告） |
| 移除按钮 | 从列表中移除该文件（不删除原文件） |

**冲突检测**：若同一批次中有两个文件生成相同的新文件名，两行均标红并在底部显示错误提示，「确认重命名」按钮禁用。

---

### 2.4 执行流程

```
选择/拖入文件
    ↓
选择重命名规则 → 实时预览新文件名
    ↓（可选）
拖拽调整顺序 / 手动编辑个别新名
    ↓
检查冲突（自动，实时）
    ↓
点击「确认重命名」→ 二次确认弹窗 → 写入磁盘
    ↓
显示执行结果（成功 N 个 / 失败 M 个）
```

> **重要**：仅点击「确认重命名」并通过二次确认后，才真正调用 `fs:renameFile` IPC 写入磁盘。预览阶段不产生任何文件操作。不支持撤销。

## 4. 新增 IPC 说明

插件需要一个现有 IPC 中没有的能力：**文件重命名**。需按「新增 IPC 三步骤」扩展：

### 4.1 新增通道：`fs:renameFile`

| 项目 | 值 |
|---|---|
| 通道名 | `fs:renameFile` |
| 参数 | `oldPath: string`, `newPath: string` |
| 返回 | `Promise<void>`（失败时 reject，携带错误信息） |

**实现步骤：**

1. **`src/main/main.ts`** — 添加 handler：
   ```ts
   ipcMain.handle('fs:renameFile', async (_e, oldPath: string, newPath: string) => {
     await fs.rename(oldPath, newPath);
   });
   ```

2. **`src/main/preload.ts`** — 在 `contextBridge.exposeInMainWorld` 中添加：
   ```ts
   renameFile: (oldPath: string, newPath: string) =>
     ipcRenderer.invoke('fs:renameFile', oldPath, newPath),
   ```

3. **`plugins/shared/bridge/src/types.ts`** — `ElectronAPI` 接口添加：
   ```ts
   renameFile(oldPath: string, newPath: string): Promise<void>;
   ```
